The Problem

Code tells you what.
Not why.

After 5 sessions, an AI without memory produces code that contradicts 70% of your original design decisions. It doesn't remember why you chose bcrypt over argon2, or cursor pagination over offset.

Without Stellar Memory
  • Re-reads all files every session (3,627+ tokens)
  • Forgets WHY decisions were made
  • Asks the same questions repeatedly
  • Switches from nanoid to UUID mid-project
  • Introduces offset pagination where cursor was used
  • Creates new error handling instead of using AppError
  • 30% code consistency by session 5
With Stellar Memory
  • Restores full context in 314 tokens
  • Remembers every decision AND the reasoning
  • Zero redundant questions across sessions
  • Consistently uses nanoid(21) everywhere
  • Follows cursor pagination pattern automatically
  • Knows AppError exists and uses it correctly
  • 95% code consistency after 5 sessions

Why Stellar Memory

The memory your
AI deserves.

Not just a key-value store. An intelligent memory system modeled after how human cognition actually works.

☾

Orbital Importance

Important memories orbit close to the Sun for instant recall. Stale knowledge drifts outward through natural decay. The half-life is 72 hours by default.

⚙

Hybrid Search

FTS5 keyword search + vector KNN search merged via Reciprocal Rank Fusion. Finds both exact matches and semantic similarities.

★

100% Local

Everything runs on your machine. SQLite database, local embeddings (all-MiniLM-L6-v2), no API keys, no cloud, no data leaving your device.

⚡

Fixed Token Budget

Context stays under 800 tokens regardless of project size. A 10,000-line codebase costs the same as a 100-line one to restore context.

How It Works

Celestial mechanics
for AI cognition.

Memories orbit a central Sun. Important ones stay close for instant recall. Forgotten ones drift to the Oort Cloud. Access pulls them back.

Core (instant recall)
Near (recently used)
Active (in-context)
Fading (drifting away)
Get Started

Three steps.
Two minutes.

01

Clone & Build

git clone https://github.com/sangjun0000/stellar-memory.git
cd stellar-memory
npm install
npm run build
02

Add to Claude Code

// .claude/settings.json
{
  "mcpServers": {
    "stellar-memory": {
      "command": "node",
      "args": ["/path/to/dist/index.js"]
    }
  }
}
03

Start Coding

That's it. Stellar Memory automatically remembers decisions, recalls context at session start, and commits state before exit. Your AI now has persistent memory.

Features

In-memory tier of 200 core memories for sub-millisecond keyword lookup.
